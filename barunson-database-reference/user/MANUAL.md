## 1. 시작하기

### 1.1 ERP란?

바른컴퍼니 ERP는 **발주 → 입고 → 재고 → 판매 → 회계**까지 회사 업무 전체를 하나의 시스템에서 관리하는 통합 관리 프로그램입니다.

### 1.2 주요 특징

- **한국/중국 발주 분리** — 국가별 흐름에 맞춰 최적화
- **법인 분리** — 바른컴퍼니 / 디디(DearDear) 데이터 완전 분리
- **자동화** — 자동발주, 이메일 자동 발송, 후공정 자동 연동
- **Slack 알림** — 발주/에러/일일 요약을 메신저로 받기

### 1.3 접속 주소

- **운영**: https://docker-manager.barunsoncard.com/c/s-c-erp/
- **브라우저**: Chrome, Edge 권장 (Safari, Firefox도 가능)

## 4. 구매/발주 (한국)

### 4.1 전체 흐름

```
자동발주 계산 → 발주대상 확인 → 일괄 저장 + 이메일 자동 발송
     ↓                                         ↓
  재고/출고량 기반                      거래처 이메일로 PDF 발송
     ↓                                         ↓
  발주 수량 자동 산정                    후공정 업체는 원재료 출고 시 자동 발송
```

### 4.2 자동발주 (핵심 기능)

**경로**: 메가메뉴 → 구매 → **자동발주**

1. 상단 국가 토글 **🇰🇷 한국** 선택 확인
2. 조회 기간 설정 (기본: 최근 3개월)
3. **🔄 계산 실행** 클릭
4. 부족한 품목 → 거래처별로 자동 그룹핑
5. 수량 직접 조정 가능 (빨간 숫자 = 긴급, 노란 = 주의)

### 4.3 발주서 저장 + 자동 발송 ⭐ (v1.0.0)

**경로**: 자동발주 페이지 → 거래처 카드 선택 → **선택 발주서 일괄 저장**

1. 저장할 거래처 체크박스 선택
2. 납기일/비고 입력
3. **일괄 저장** 버튼 클릭
4. 확인 다이얼로그:
   ```
   [한국 발주 저장 + 자동 발송]
   • 원재료 거래처: 5개 → 저장 후 즉시 이메일 발송
   • 후공정 거래처: 2개 → 저장만 (원재료 출고 시 자동 발송)
   진행하시겠습니까?
   ```
5. **확인** → 자동으로:
   - 발주서 DB 저장
   - 거래처 이메일로 PDF 발송 (한국 원재료만)
   - Google Sheet 동기화
   - Slack 알림 (설정 시)
6. 결과 알림:
   ```
   5개 거래처 발주서 모두 저장 완료!
   [이메일 자동 발송]
   ✅ 발송 성공: 4건 (대한통상, 두성종이, 삼원특수지, 서경)
   ⚠️ 미발송: 1건
     - (업체명) (업체 이메일 미등록)
   ```

### 4.4 발주현황 관리

**경로**: 메가메뉴 → 구매 → **발주현황**

- 상태별 필터: 대기/발송/확인/수령중/완료/취소
- PO 클릭 → 상세 (품목/거래처/이력)
- **재발송** 버튼 — 이메일 재전송
- **취소** 버튼 — PO 취소 + Google Sheet 취소선 적용

### 4.5 업체 포털 (거래처 쪽)

거래처는 이메일 링크 클릭 → 별도 로그인 없이 **발주 확인**, **출고 처리** 가능
→ 원재료 업체가 "출고 처리" 하면 **같은 날짜의 후공정 업체에 자동 이메일 발송**

## 5. 구매/발주 (중국)

### 5.1 중국 발주 특징

- 이메일 체계 X → **엑셀 다운로드 후 수동 전달**
- 交期表(납기표) 기반 관리
- 선적(BL) 추적

### 5.2 중국 발주 흐름

1. 상단 국가 토글 **🇨🇳 중국** 선택
2. 자동발주 → 수량 조정 → **일괄 저장**
3. 저장 후 **중국 발주서 엑셀 다운로드** 안내
4. 엑셀 다운로드 → 담당자에게 전달 (위챗/메일)

### 5.3 중국선적 관리

**경로**: 메가메뉴 → 중국 → **중국선적**

- BL 번호 등록
- 선적 → 통관 → 입고 단계별 상태 관리
- 도착 예정일 달력 표시

## 6. 재고 관리

### 6.1 재고 현황

**경로**: 메가메뉴 → 재고 → **재고현황**

- XERP 실시간 재고 조회
- 품목별 현재고/가용재고/예약재고
- 부족 품목 필터 (안전재고 미달)

### 6.2 입출고 현황

**경로**: 메가메뉴 → 재고 → **입출고 현황**

- 일자별 입고/출고 내역
- XERP 데이터 자동 동기화
- 엑셀 다운로드 가능

### 6.3 Lot 추적

**경로**: 메가메뉴 → 재고 → **Lot 추적**

- 원재료 Lot 번호로 이력 추적
- 품질 문제 발생 시 **역추적** 가능

### 6.4 안전재고 관리

**경로**: 메가메뉴 → 재고 → **안전재고**

- 품목별 안전재고 설정
- 미달 시 자동발주에 반영됨

## 11. 자주 묻는 질문

### Q1. 발주서를 저장했는데 이메일이 안 갔어요

**A**: 확인 사항:
1. **한국** 발주인지 확인 (중국은 이메일 발송 안 함)
2. **원재료** 거래처인지 확인 (후공정은 원재료 출고 시 자동 발송)
3. 거래처 관리에서 해당 업체 이메일이 등록되어 있는지 확인
4. 결과 알림창의 "⚠️ 미발송" 사유 확인

### Q2. 같은 품목을 오늘 또 발주하려는데 "중복 발주" 오류가 나요

**A**: 오늘 동일 거래처에 같은 품목 조합으로 이미 발주가 있으면 차단됩니다.
- 해결: 기존 PO를 취소하거나, 품목을 다르게 묶어서 발주

### Q3. 국가 토글을 바꿔도 데이터가 안 바뀌어요

**A**: 페이지 새로고침(F5) 또는 해당 페이지 메뉴 재클릭

### Q4. Google 로그인이 안 돼요

**A**: 브라우저 쿠키 허용 확인 → 서드파티 쿠키 차단 시 작동 불가

### Q5. 바른손 데이터와 DD 데이터가 섞여 보여요

**A**: 상단 **법인 토글** 확인. `바른컴퍼니` / `디디` 완전 분리 정책이므로 반드시 해당 법인 선택 후 조회

## 12. 문제 해결

### 12.1 서버 응답이 없을 때

1. 우측 상단 🔔 알림 → 연결 상태 확인
2. F12 → Console 탭 → 에러 메시지 확인
3. 관리자(백승찬)에게 문의

### 12.2 데이터가 이상할 때

- **XERP 동기화**: 매일 오전 9:30 자동 실행
- 수동 동기화 필요 시 → 관리자에게 요청

### 12.3 이메일 발송 실패 시

**발주현황** → 해당 PO 클릭 → **재발송** 버튼
→ 그래도 안 되면 거래처 이메일 주소 재확인

### 12.4 긴급 문의

- 시스템 담당: **백승찬** (`seungchan.back@barunn.net`)
- 서버 에러 시 Slack `#erp-alerts` 채널에 자동 알림 (설정 시)
