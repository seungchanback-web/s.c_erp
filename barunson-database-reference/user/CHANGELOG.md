## [1.0.9] - 2026-04-10

### 🎉 Added — 발주 프로세스 개선 + 거래처 포탈 강화
- **후공정 "사용안함(패스)" 옵션** — 해당 공정이 필요 없는 품목의 후공정 건너뛰기
- **후공정 "기타업체 직접입력"** — 등록된 업체 외 직접 입력 가능 (드롭다운에서 선택 시 입력창 활성화)
- **발주서 완료/복원 기능** — 발주서 관리에서 완료 마킹 + 복원 가능 (PATCH /api/po-drafts/:id/status)
- **발주서 법인 구분 표시** — DD=붉은색 테두리/뱃지, 바른컴퍼니=기본
- **거래처 포탈 재고/긴급도 표시** — 원재료 품목에 현재 재고 기반 긴급도 (🚨긴급 / ⚠부족 / 안전)
- **거래처 포탈 법인 뱃지** — PO 카드에 "DD"(붉은색) / "바른"(파란색) 법인 식별 표시
- **한국 리드타임 기본값 20영업일** — 기존 5일 → 20일로 변경 (GUIDE_KR.leadTimeMat)
- **필요입고일 계산 리드타임 연동** — 기존 5영업일 고정 → GUIDE 리드타임(영업일) 기준 계산
- **발주필요일 날짜 표시** — "D+N 지연" 텍스트 대신 실제 날짜 표시 (색상으로 긴급도 구분)
- **pg-adapter pkMap 대량 보강** — batch_inspections, work_order_logs 등 16개 테이블 PK 매핑 추가

### 🐛 Fixed
- **CRITICAL: "발주 확인" 버튼** — 불량관리 처리 발주에서 존재하지 않는 'po' 페이지로 이동 → 'po-list' 페이지로 수정 + 검색 자동 필터
- **PostgreSQL batch_inspections INSERT 500에러** — pkMap에 insp_id 누락으로 id 컬럼 참조 실패
- **PostgreSQL work_order_logs INSERT 500에러** — pkMap에 log_id 누락으로 id 컬럼 참조 실패

## [1.0.8] - 2026-04-10

### 🎉 Added — DD 법인 재고/출고 완전 분리 (구조 개편)
- **BHC(디얼디어) DB 연동** — DD 법인 재고 및 출고 데이터를 **BHC DB의 `mmInventory`/`mmInoutItem`** 에서 직접 조회 (SiteCode `BHC2`)
- **`/api/xerp-inventory` 법인별 DB 분기**:
  - `company=barunson` → XERP DB / SiteCode `BK10` (바른손 창고, 기존)
  - `company=dd` → **BHC DB / SiteCode `BHC2`** (디얼디어 창고, 신규)
  - `company=all` → 두 DB 병렬 조회 후 병합 (Promise.all)
- 응답 품목마다 `legal_entity` / `_invSource` / `_siteCode` 필드 동봉 — 프론트 필터링 정확도 향상
- **발주서 작성 페이지에 법인 필터 셀렉트 추가** — 바른컴퍼니/디디 전환
- `getPOTargetItems` / 생산지 탭 배지에 `_poCurrentEntity` 기준 필터 로직 추가
- 메가메뉴 `create-po` 항목의 `companies:['barunson']` → `['barunson','dd']` 로 확장, `origins`에 `DD` 추가
- BHC 내 DD 품목 705개 고유, 재고>0 472개 확인 (수정 전 발주제안 탭에 1건만 노출되던 문제 원천 해결)

### 🐛 Fixed
- **DD 품목 `origin='한국'` 강제 UPDATE 제거** (`serve_inv2.js:1182`) — 원래 생산지가 소실되어 발주서 작성 한국 탭에서 바른손과 섞이던 구조적 버그 수정. `legal_entity='dd'` 마킹만 수행하고 `origin` 원본 보존

### 📝 Notes
- BHC 접속은 기존 `readonly_user` (`DB_USER`) 로 가능 — XERP 계정(`XERP_DB_USER`)은 BHC 권한 없음, Azure SQL 권한 매핑 차이
- BHC pool은 on-demand 생성(`new sql.ConnectionPool` → `close`) — 장기 커넥션 필요 없음, bar_shop1 pool과 동일 패턴

## [1.0.7] - 2026-04-10

### ✨ Added
- **품목 엑셀 일괄 업로드 미리보기 모달** — 업로드 버튼 클릭 시 바로 저장되지 않고, 신규/기존/오류 건수를 먼저 보여주는 미리보기 모달이 뜸. 사용자가 명시적으로 선택:
  - **🆕 신규만 등록**: 새로운 품목만 INSERT, 기존 품목은 손대지 않음 (안전)
  - **🔄 전부 반영**: 신규 INSERT + 기존 엑셀 값으로 덮어쓰기 (빈 칸은 기존값 유지)
  - **취소**: 아무것도 안 함
- 상위 5건씩 상세 내역 접이식(`<details>`) 표시 + 오류 행 별도 카운트
- 신규 API: `POST /api/products/bulk/preview` (저장 없이 분석만)
- `POST /api/products/bulk` 에 `mode` 파라미터 추가 (`upsert`|`insert_only`)
- **발주 3단계 breadcrumb** — 발주서 작성 페이지 상단에 흐름 시각화 추가:
  `[1. 발주제안 (재고현황)] → 【2. 발주서 작성】 → [3. 발주현황]`
  각 단계 클릭으로 이동 가능

### 🔧 Changed
- **메뉴/버튼 이름 정리** — "발주대상"이라는 모호한 용어를 역할 기반으로 재정의:
  - 구매관리 메가메뉴 `발주대상` → **`발주서 작성`** (설명도 "선택 품목으로 PO 생성/발송"으로 명확화)
  - 재고현황 > 발주제안 탭 `[발주대상]` 버튼 → **`[발주서 작성 →]`** (툴팁: 선택한 품목을 발주서 작성 페이지로 이동)
  - create-po 페이지 타이틀 → `✍️ 발주서 작성`
  - 페이지명 매핑(`PAGE_NAMES`)도 동기화
- 재고현황 > 발주제안 탭 = "시스템이 제안하는 품목 선택" 단계, 구매관리 > 발주서 작성 = "실제 PO 작성/발송" 단계로 역할 분리가 이름에서 바로 드러남

## [1.0.5] - 2026-04-10

### 🐛 Fixed
- **품목관리에는 등록되어 있는데 재고현황에 DD 품목이 안 보이던 문제** — `/api/xerp-inventory`의 생산지 필터가 `origin = 'DD'` 하드코딩이어서 생산지='한국'으로 저장된 우리 DD 품목들이 재고현황 DD 탭에서 누락되었음. 품목코드 prefix(`DD*`) 기반으로 필터 전환하여 `origin = '한국'`인 DD 품목도 정상 노출되도록 수정
- **재고현황 비활성 품목 필터** — DD 탭에서는 `status='inactive'`인 DD 품목도 함께 노출되도록 예외 처리

## [1.0.3] - 2026-04-10

### 🐛 Fixed
- **재고현황 페이지 디디 필터 작동 안 하던 문제** — `isDD` 판단이 `생산지==='DD'` 하드코딩이라 우리 DD 품목(생산지='한국')은 항상 false였음. 품목코드 prefix(`DD*`) 기반으로 통일 + `legal_entity` 필드도 함께 체크

## [1.0.2] - 2026-04-10

### 🐛 Fixed
- **디디 품목이 품목관리 디디 탭에서 완전히 비어 보이던 심각한 버그 수정**
  - 운영 PG에 `products.legal_entity` 컬럼이 DB 권한 부족으로 `ALTER TABLE`이 silent fail한 상태였음 (`_hasEntity.products = false`)
  - 이 때문에 `GET /api/products` 응답에 `legal_entity` 필드가 아예 없었고, 프론트가 기본값 `'barunson'`으로 처리 → 디디 탭 필터에서 모든 품목이 탈락
  - 앱 레벨 fallback 추가: 컬럼이 없는 환경에서는 품목코드 prefix로 가상 매핑 (`DD*` → `dd`, 나머지 → `barunson`)
  - entity 파라미터가 있는데 DB에서 필터를 못 건 경우 메모리에서 한 번 더 필터링
  - DB 권한 문제가 해결되면 컬럼 기반으로 자동 전환됨

## [1.0.1] - 2026-04-09

### 🐛 Fixed
- **DD 한국 품목이 품목관리 화면에 보이지 않던 문제 수정**
  - 디디(DD) 법인의 엑셀 업로드/수동 등록 시 생산지 값이 `예지가`, `코리아`, `마커엘엔피`, `코리아-예지가` 등 변형값으로 저장되는 경우 품목관리 "한국" 탭 필터(`origin === '한국'`)에 잡히지 않아 등록은 성공했지만 리스트에 표시되지 않던 문제
  - `renderProductMgmt`에서 DD 법인에 한해 한국계 변형 생산지값(`코리아*`, `예지가*`, `마커*`)을 "한국" 탭으로 정규화하여 표시
  - 원본 생산지 값은 품목명 옆에 작은 핑크 배지로 함께 표시해 구분 가능하도록 함
- **엑셀 업로드 시 법인 누락 방지** — `handlePMExcel`이 현재 선택된 탭(`pmEntity`)을 자동으로 `legal_entity`에 세팅. 엑셀에 `법인` 컬럼이 있으면 `바른손/디디/dd/barunson` 자동 인식
- **서버측 생산지 정규화 방어 로직 추가** — `POST /api/products`, `POST /api/products/bulk`에서 변형 생산지값(`예지가/코리아/마커엘엔피`)을 DB 저장 직전 `한국`으로 자동 변환

### 🔧 Changed
- **품목등록 엑셀 템플릿 개편**
  - `법인` 컬럼 추가 (바른손/디디)
  - `입력규칙` 시트 추가: 각 컬럼의 필수 여부, 허용값, 주의사항 문서화
  - 현재 선택된 법인에 따라 템플릿 예시행 자동 전환 (`바른손_품목등록_양식.xlsx` / `디디_품목등록_양식.xlsx`)
  - 생산지 허용값을 `한국/중국/더기프트` 3개로 명시 — 디디도 `예지가/코리아/마커엘엔피` 사용 금지 안내

## 📌 버전 규칙

Semantic Versioning (SemVer) 준수:

- **MAJOR** (x.0.0) — 전체 흐름 변경, 호환성 깨지는 변경
- **MINOR** (0.x.0) — 기능 추가, 하위 호환
- **PATCH** (0.0.x) — 버그 수정, 문구 개선

1. `serve_inv2.js` / `app.html` 등 수정
2. 기능 변화가 있다면 `MANUAL.md` 해당 섹션 업데이트
3. `CHANGELOG.md` 최상단에 새 버전 항목 추가 (`Added / Fixed / Changed / Removed`)

5. `serve_inv2.js`의 `APP_VERSION` 상수 업데이트 (있다면)
