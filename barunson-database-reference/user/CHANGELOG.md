## [1.0.1] - 2026-04-09

### 🐛 Fixed
- **DD 한국 품목이 품목관리 화면에 보이지 않던 문제 수정**
  - 디디(DD) 법인의 엑셀 업로드/수동 등록 시 생산지 값이 `예지가`, `코리아`, `마커엘엔피`, `코리아-예지가` 등 변형값으로 저장되는 경우 품목관리 "한국" 탭 필터(`origin === '한국'`)에 잡히지 않아 등록은 성공했지만 리스트에 표시되지 않던 문제
  - `renderProductMgmt`에서 DD 법인에 한해 한국계 변형 생산지값(`코리아*`, `예지가*`, `마커*`)을 "한국" 탭으로 정규화하여 표시
  - 원본 생산지 값은 품목명 옆에 작은 핑크 배지로 함께 표시해 구분 가능하도록 함
- **엑셀 업로드 시 법인 누락 방지** — `handlePMExcel`이 현재 선택된 탭(`pmEntity`)을 자동으로 `legal_entity`에 세팅. 엑셀에 `법인` 컬럼이 있으면 `바른손/디디/dd/barunson` 자동 인식
- **서버측 생산지 정규화 방어 로직 추가** — `POST /api/products`, `POST /api/products/bulk`에서 변형 생산지값(`예지가/코리아/마커엘엔피`)을 DB 저장 직전 `한국`으로 자동 변환

### 🔧 Changed
- **품목등록 엑셀 템플릿 개편**
  - `법인` 컬럼 추가 (바른손/디디)
  - `입력규칙` 시트 추가: 각 컬럼의 필수 여부, 허용값, 주의사항 문서화
  - 현재 선택된 법인에 따라 템플릿 예시행 자동 전환 (`바른손_품목등록_양식.xlsx` / `디디_품목등록_양식.xlsx`)
  - 생산지 허용값을 `한국/중국/더기프트` 3개로 명시 — 디디도 `예지가/코리아/마커엘엔피` 사용 금지 안내

## [1.0.0] - 2026-04-09

### 🎉 첫 공식 버전 (사용 설명서 v1 발간)

#### ✨ Added (신규)
- **사용 설명서 발간** — `MANUAL.md` / ERP 내부 `사용설명서` 페이지 추가
- **한국 원재료 발주 자동 이메일 발송** — `saveAllPOs` 저장 시 한국 원재료 PO는 자동으로 거래처에 이메일 발송
- **공용 관리자 계정** — `admin / 1234` 추가 (팀 공용 접속용)
- **Slack 알림 시스템**:
  - 📮 수동 발주 완료 요약 (`POST /api/slack/manual-po-notify`)
  - 🔗 원재료→후공정 자동 이메일 체인 알림 (성공/이메일없음/발송실패/매칭없음)
  - 🔴 서버 크래시, uncaughtException, unhandledRejection 자동 알림
  - 📋 매일 오전 9시 자동발주 요약
  - 에러 중복 방지 (5분 dedupe)
- **발주 엑셀 컬럼 확장** — 일평균 출고량, 예상한달 출고량, 현재고, 발주수량 추가

#### 🐛 Fixed (버그 수정)
- **saveAllPOs 응답 파싱 버그** — `ok()` 헬퍼의 `{ok:true, data:{...}}` 래퍼를 프론트에서 직접 `d.po_id`로 접근하던 문제 수정
- **법인 분리 완전 적용** — XERP(바른손)와 DD(뚜비뚜비) 데이터 완전 분리

#### 🔧 Changed (변경)
- 기존 2단계 발주 흐름(저장 → 수동 발송)을 **1단계**(저장 + 자동 발송)로 통합
- 중국 발주는 기존 흐름(엑셀 다운로드) 유지
- 후공정 발주는 기존 흐름(원재료 출고 시 자동 체인) 유지
