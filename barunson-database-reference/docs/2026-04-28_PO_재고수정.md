## 원인 분석

### A. DD 재고 안 불러와짐
- `barunson-database-reference/user/serve_inv2.js:5895` — `validCodeSet` 가 `productCodes.toUpperCase()` 로만 생성. XERP `mmInventory.ItemCode` 와 로컬 `products.product_code` 형식이 어긋날 때(예: 로컬 `DDC_0211`, XERP `DDC0211`) 매칭 실패.
- 결과: 4월 24일 resync 기준 디디 54개 중 **16개 unmatched** (`DDC_0211, DDC_0213, DDC_1201~06, DDC_2217` 등). monthly=0 으로 들어가니 파생 컬럼(가용일수=현재고/일평균, 안전재고=일평균×G.safetyMonths) 전부 "-" 또는 0.
- 가용일수/안전재고/미입고/예상재고 계산 로직 자체는 `app.html:7727-7754` 에 이미 존재. 데이터만 정상이면 자동 채워짐 → **백엔드 매칭만 고치면 됨**.

### B. 발주현황 후공정 라인 컬럼 부족
- `app.html:16957` 헤더 = `제품코드 / 후공정업체 / 공정 / 수량(R) / OS번호 / 입고현황 / 입고처` (7개)
- 원재료 라인(16956)은 9개 — 후공정에는 **원자재코드, 원재료용지명, 절, 거래처** 컬럼이 없었음

### C. 입고처 = "바른손" 하드코딩
- `app.html:17005` (수정 전) — 후공정 행 입고처 셀이 `바른손` 으로 직접 박혀 있어, 다음 후공정 업체로 넘어가는 PO 도 자사창고로 표시됨

### D. 봉투가공 위치
- `app.html:7506, 14864, 14104, 17244` 다섯 군데 PROCESS_TYPES_POOL/폴백 배열에 `'재단','인쇄','박/형압','톰슨','봉투가공',...` — 봉투가공이 5번째
- `app.html:7545` 클라이언트 정렬, `serve_inv2.js:636, 5623` 백엔드 ORDER BY — step_order 만 사용하고 봉투가공 강제 후미 규칙 없음

### E. 동기화 속도
- 호출 흐름: `POST /api/sync/xerp-inventory` → 백그라운드에서 `/api/xerp-inventory?refresh=1&company=all` self-call → barunson + dd 병렬(`Promise.all`) → 200개 단위 UPSERT
- 병렬화는 이미 적용됨(serve_inv2.js:6091-6094). 매칭 정규화 후 unmatched 가 줄면 효과적 동기화량 증가.

## 수정 내용

### 1) DD 매칭 정규화 — `serve_inv2.js`

**Before**
```js
const validCodeSet = new Set(productCodes.filter(c => /^[A-Za-z0-9_\-]+$/.test(c))
                                          .map(c => c.toUpperCase()));
// ... mmInventory loop
const code = (row.item_code || '').trim().toUpperCase();
if (!code || !validCodeSet.has(code)) continue;
```

**After**
```js
const normalizeCode = c => (c || '').toUpperCase().replace(/[_\-\s]/g, '');
const normMap = {};
for (const c of productCodes) {
  if (!/^[A-Za-z0-9_\-]+$/.test(c)) continue;
  const k = normalizeCode(c);
  if (k && !normMap[k]) normMap[k] = c.toUpperCase();
}
const resolveLocal = xerpCode => {
  const upper = (xerpCode || '').toUpperCase();
  if (validCodeSet.has(upper)) return upper;       // 원본 매칭 우선
  const norm = normalizeCode(upper);
  return norm ? (normMap[norm] || null) : null;    // 정규화 폴백
};
// ... mmInventory loop
const rawCode = (row.item_code || '').trim().toUpperCase();
const code = resolveLocal(rawCode);
if (!code) continue;
if (code !== rawCode) _normMatched++;
```

같은 처리를 `mmInoutItem` (출고) 쿼리에도 적용. 정규화로 매칭된 건수는 로그(`정규화 보정 N건`)로 추적.

### 2) 봉투가공 항상 마지막 — 5군데 동시 수정

**Before** (대표)
```js
const PROCESS_TYPES_POOL = ['재단','인쇄','박/형압','톰슨','봉투가공','단면접착','우찌누끼','접지','코팅'];

ppvByCode[code].sort((a,b) => (a.step_order||1) - (b.step_order||1));

await db.prepare('SELECT ... ORDER BY product_code, step_order').all();
```

**After**
```js
const PROCESS_TYPES_POOL = ['재단','인쇄','박/형압','톰슨','단면접착','우찌누끼','접지','코팅','봉투가공'];

ppvByCode[code].sort((a,b) => {
  if (a.process_type==='봉투가공' && b.process_type!=='봉투가공') return 1;
  if (b.process_type==='봉투가공' && a.process_type!=='봉투가공') return -1;
  return (a.step_order||1) - (b.step_order||1);
});

await db.prepare("SELECT ... ORDER BY product_code, "
  + "CASE WHEN process_type='봉투가공' THEN 1 ELSE 0 END, step_order").all();
```

`getPostCols()` 함수에도 `sortBongtuLast` 헬퍼 적용 — DB 에 옛 순서로 들어와 있어도 화면 단에서 항상 마지막으로 밀림.

### 3) 후공정 PO 라인 컬럼 확장 — `app.html`

**Before** (16957 + 16998-17006)
```
헤더: 제품코드 / 후공정업체 / 공정 / 수량(R) / OS번호 / 입고현황 / 입고처
행:   ...                                                  / "바른손"  ← 하드코딩
```

**After**
```
헤더: 제품코드 / 거래처 / 원자재코드 / 원재료용지명 / 절 / 공정 / 수량(R) / OS번호 / 입고현황 / 입고처
행:   it.product_code / po.vendor_name / info['원자재코드'] / info['원재료용지명']
      / info['절'] / it.process_type / 수량 / OS / 입고현황
      / nextStep?.vendor || '바른손'   ← 다음 공정 업체 lookup
```

`info._steps` (이번 커밋에서 추가) 에서 현재 공정의 다음 단계를 찾아 그 업체명을 입고처로 표시. 마지막 단계면 자사창고('바른손') 폴백.

### 4) `productInfo._steps` 보강 — `app.html:7575-7580`

**Before**
```js
for(const[code, steps] of Object.entries(ppvByCode)){
  if(!productInfo[code]) productInfo[code]={};
  steps.forEach(s=>{ productInfo[code][s.process_type]=s.vendor_name; });
}
```

**After**
```js
for(const[code, steps] of Object.entries(ppvByCode)){
  if(!productInfo[code]) productInfo[code]={};
  steps.forEach(s=>{ productInfo[code][s.process_type]=s.vendor_name; });
  productInfo[code]._steps = steps.map(s => ({
    process: s.process_type, vendor: s.vendor_name, step: s.step_order||1
  }));
}
```
